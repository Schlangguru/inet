Throughput Example

Goals

This example analizes how the application level throughput changes in the function of the nominal bitrate in a 802.11a network.

The model

Application level throughput

802.11 modes are denoted by the nominal data bitrate (e.g. 54 Mbps). However, the bitrate available for an application is less because of various protocol overheads, like preamble, physical and mac headers. In this model, the actual application level throughput is measured.

Configuration

The network contains two wirelessHosts, with a distance of 1 meter. The sourceHost sends a UDP stream of 1000 byte packets to destinationHost in ad-hoc mode. The UDP application in sourceHost is configured to saturate the channel when it is transmitting with the fastest bitrate. There are no lost packets, because the hosts are close to each other. The path loss model is free space path loss.

The parameter study iterates all 802.11 mode a bitrates (6, 9, 12, 18, 24, 36, 48, 54 Mbps). Each simulation runs for 1 second, and the throughput is averaged for this interval.

Results

The results are compared to theoretical computed values for the throughput. The frame exchange duration can be calculated from the nominal bitrate and the payload size. For example, the excel sheet here can be used for this purpose. It takes the data frame, the ack frame, and the interframe gap between them into account. By assuming an average back off time that is half of the minimal contention window, the theoretical throughput can be calculated.
